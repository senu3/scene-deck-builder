# ADR-0003: Timeline構造変更の Command 境界を固定する

## Status
Accepted (2026-02-17)

## Context
- Gate 6 は「Timeline 構造変更は Command 経由」を要求するが、実装上は例外扱いが曖昧だと運用が崩れる。
- すべての状態変更を Command 化すると、ロード/復元/初期化のようなシステム起点処理まで過剰に複雑化する。
- 実際の破壊的な回帰は、ユーザー操作起点の構造変更で発生しやすい。

## Decision
- Command 必須対象を「ユーザー操作起点の timeline 構造変更」に限定する。
  - 対象: scene/cut/group の追加・削除・移動・並び替え。
- 例外として、次は Command 対象外を許可する。
  - プロジェクトのロード/復元。
  - 移行・正規化・初期化（autosave 反映を含む）。
  - テスト初期化や fixture 構築。
- 例外経路では、構造整合（`sceneOrder`, `cut.order` 連番）を満たす後処理を必須にする。

## Consequences
- レビュー基準が明確化され、Command 強制の適用範囲を機械的に判断しやすくなる。
- ロード・復元パスは実装自由度を保ちつつ、整合性検証を強化する方針に寄せられる。
- Gate 6 の残課題は「境界違反の検出」と「例外経路の整合チェック追加」に限定できる。
